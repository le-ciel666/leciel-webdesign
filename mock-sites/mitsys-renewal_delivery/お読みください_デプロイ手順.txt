 株式会社MITシステム研究所 コーポレートサイト 一式
 デプロイ手順書（情報システムご担当者さま向け）
================================================================

このZIPには、Webサイトを公開するために必要なファイル一式が含まれています。
特別なプログラム言語の知識がなくても、Webサーバーにアップロードするだけで公開できます。

----------------------------------------------------------------
1. ZIPの中身
----------------------------------------------------------------

  mitsys-website/
   ├─ お読みください_デプロイ手順.txt   ← このファイル（※アップロード不要）
   └─ site/                              ← この「中身」を公開ディレクトリへ
        ├─ index.html                    … トップページ
        ├─ contact/                      … お問い合わせ
        ├─ download/                     … 資料請求／資料ダウンロード
        ├─ privacy/                      … 個人情報の取り扱いについて
        ├─ quickcrm-renewal/             … QuickCRM 製品サイト
        └─ assets/                       … 画像・ロゴ・PDF など

  ※「お読みください_デプロイ手順.txt」は社内向けの説明書です。
    Webサーバーにはアップロードしないでください。

----------------------------------------------------------------
2. 公開（デプロイ）手順 ― ルート公開
----------------------------------------------------------------

  公開先URL : https://www.mitsys.co.jp/

  手順:
   (1) ZIPを解凍します。
   (2) 「site」フォルダの【中身】（index.html や assets フォルダなど）を、
       Webサーバーの公開ディレクトリ直下（例: /public_html/ や /htdocs/ や /www/）に
       そのままアップロードします。
       ※「site」フォルダごとではなく、site の中身を置いてください。
       　 正: https://www.mitsys.co.jp/index.html
       　 誤: https://www.mitsys.co.jp/site/index.html
   (3) ブラウザで https://www.mitsys.co.jp/ を開き、表示を確認します。

  フォルダ構成（相対パス）はそのまま維持してください。
  フォルダ名やファイルの配置を変えると、ページ間リンクや画像が表示されなくなります。

----------------------------------------------------------------
3. 動作要件
----------------------------------------------------------------

  ・静的サイト（HTML / CSS / JavaScript / 画像）です。
  ・データベースや特別なサーバー設定は不要です。
  ・文字コードは UTF-8 です。
  ・対応ブラウザ: 最新の Chrome / Edge / Safari / Firefox、各スマートフォン標準ブラウザ。

----------------------------------------------------------------
4. 【重要】お問い合わせ・資料請求フォームについて（公開前に必ずご確認ください）
----------------------------------------------------------------

  本サイトのフォームは、画面（入力UI）のみが用意された状態です。
  送信内容をメール送信・保存する「サーバー側の処理（バックエンド）」は
  実装されていません。

  ▼ このまま公開した場合
     利用者がフォームを送信しても、入力内容はどこにも届きません。
     （画面上は「送信完了」と表示されますが、実際には送信されません）

  ▼ 公開前に必要な対応
     送信内容を受信できるよう、サーバー側の送信処理（例: PHPでのメール送信、
     フォーム送信サービスの利用など）を実装し、各フォームの送信先を設定してください。

  ▼ 対象フォーム（4件）
     ・/contact/                      （お問い合わせ）
     ・/download/                     （資料請求）
     ・/quickcrm-renewal/contact/     （QuickCRM お問い合わせ）
     ・/quickcrm-renewal/download/    （QuickCRM 資料請求）

  ※フォームの送信処理の実装については、制作元（下記）でも対応可能です。
