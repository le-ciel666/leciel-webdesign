01  /  Background

なぜ、作り直すのか。

QuickCRM は「お客様自身が、思う業務がそのまま動く環境を構築できる」ことを強みとした CRM です。 ただし、その強みは、テキストと箇条書きだけでは伝わりにくく、競合のローコード CRM・SaaS と差別化しにくい性質を持っています。

リニューアル前の資料は、1 枚あたりに情報が密に詰まっており、検討者がポイントを掴むまでに時間を要しました。 そこで今回は 「読む資料」から「見て分かる資料」へ。 画面イメージ・図解・モックアップを軸に、QuickCRM の世界観をひと目で伝わる構成へ作り直します。

設計方針：テキストで説明するよりも、視覚で読ませる。
サービスの個性「自分で作れる CRM」を、ビジュアルとして体現する。
02  /  Brand Color

サービスカラーを、資料の主役に。

旧資料のフッターのみで使われていたコーポレートグリーン（#00985E）を、ビジュアル全体の主役へ昇格しました。 補助色としてライムグリーン・フォレストグリーン・ゴールドを限定的に使用し、信頼感（深いグリーン）と動き（明るいグリーン・ゴールド）が共存しています。

Primary エメラルド #00985E
Accent ライム #63A537
Deep フォレスト #1E6F43
Highlight ゴールド #FFC000
また、ロゴマーク代わりに 「Q」マークをデザインし、各スライドのヘッダー・タブ・ダッシュボードの一部などに繰り返し配置。 資料全体を通じてブランドの一貫性を保ち、QuickCRM をしっかり記憶に残るよう設計しています。
03  /  Design Principles

3 つの設計原則。

01

絵で読ませる

各スライドに必ず 1 つ以上のイラスト・図解・モックアップを配置。テキストでは伝わりにくい「自由度」「カスタマイズ性」「業務フローの全体像」を、視覚として一発で理解できる構成にします。

02

リアルな UI を見せる

画面作成方法の説明（自由配置・テンプレート・Excel）は、実際の操作画面を模した UI モックアップで表現。「自分でその画面を作る」というユーザー体験を、資料の中で疑似体験してもらいます。

03

1 スライド・1 メッセージ

旧資料の 6 枚を 14 枚へ拡張し、1 枚あたりの情報密度を意図的に下げました。スピーカーが話す内容と画面が一致しやすくなり、商談での説明がしやすくなります。

05  /  Slide Structure

14 枚で読む、QuickCRM の全体像。

ストーリーは 「直感 → 強み → 機能 → 作り方 → 運用」の流れ。
導入を検討するお客様が、サービスを理解 → 自分の業務に当てはめてイメージ → 安心するまでを、自然に追体験できる順序にしています。

06  /  Expected Outcomes

この資料で、実現したいこと。

商談での説明がしやすくなる

1 スライド 1 主張に整理されたことで、スピーカーが「次に何を話す」かを意識せずに済み、間延びが防げます。

サービスの世界観が伝わる

ブランドカラーと Q マークの一貫使用で、「QuickCRM = 緑のあのサービス」という記憶が残ります。

「自分で作れる」を疑似体験できる

UI モックを通じて、検討者がサービスの実画面を頭の中でシミュレーションでき、導入後のイメージを具体化します。

更新・展開がしやすい

スライドの型（章扉 / 図解 / モック / フロー）が確立されているため、機能追加時の差し替えや、業種特化版の派生制作にも転用しやすい構造です。
