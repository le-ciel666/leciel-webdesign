# 法務チーム（LE CIEL Webデザイン）

## チームミッション

ルシエルWebデザインが安全・安心に事業を継続できるよう、すべての取引・契約・権利関係を守る。トラブルが起きてからではなく、起きる前に防ぐことが最大の役割。

## 専門領域

- 業務委託契約書の作成・レビュー
- 著作権・知的財産権の管理
- プライバシーポリシー・特定商取引法表記の整備
- クライアントとのトラブル予防と初期対応
- 免責事項・キャンセルポリシーの設計
- クラウドワークス・ランサーズの規約対応

## 契約書テンプレート（クラウドワークス案件用）

クラウドワークスの場合は「仮払い」システムにより取引保護があるが、クラウドワークス外での直接取引には必ず書面を交わす。

### 業務委託契約書の必須記載事項

```
1. 業務内容の定義
   - 制作物の具体的な仕様（ページ数・機能・対応デバイス）
   - 含まれないもの（ドメイン代・サーバー代・写真撮影など）

2. 報酬と支払い条件
   - 金額（税込）
   - 支払いスケジュール（着手金50% + 納品時50%を推奨）
   - 支払い方法（銀行振込）
   - 支払い期限（請求書受領後14日以内）

3. 納品物と著作権
   - 納品物の定義（HTMLファイル一式等）
   - 著作権の帰属（入金確認後にクライアントへ移転）
   - 使用ライセンス（フリー素材の利用条件を明記）

4. 修正・変更
   - 無料修正の範囲と回数（2回まで）
   - 追加修正の費用（5,500円/回〜）
   - 修正依頼の期限（納品後30日以内）

5. キャンセルポリシー
   - 着手前キャンセル：着手金の返金なし
   - 制作中キャンセル：進捗に応じた費用請求
   - 納品後キャンセル：返金不可

6. 免責事項
   - 検索順位の保証はしない
   - 売上・問い合わせ件数の保証はしない
   - サーバー障害・ドメイン失効による損害は責任外

7. 秘密保持
   - 制作中に知り得たクライアント情報を第人に開示しない
```

## 著作権管理ルール

### 制作物の著作権フロー

```
制作中 → 著作権はルシエルWebデザイン（相川典義）に帰属
   ↓
入金確認後 → クライアントへ著作権譲渡
   ↓
ポートフォリオ掲載権 → ルシエルは事前に書面で許可を得る
```

### 使用素材の著作権確認チェックリスト

| 素材の種類 | 確認事項 |
|-----------|---------|
| フリー画像（Pexels・Unsplash） | 商用利用可・帰属表示不要か確認 |
| Google Fonts | OFLライセンス確認（商用利用可） |
| アイコン（Font Awesome等） | ライセンスプラン確認 |
| クライアント提供素材 | 第三者の著作物でないか確認 |
| 参考サイトの模倣 | 著作権侵害にならないレベルか判断 |

**絶対にやってはいけないこと**
- 他サイトの画像を無断でコピーして使用
- 有料フォントを無断で使用
- 競合サイトのコードをコピー

## プライバシーポリシー・特定商取引法表記

### ルシエルWebデザイン自社サイト用 必須ページ

**プライバシーポリシーに記載すべき内容**
- 収集する個人情報の種類（問い合わせフォームの入力情報等）
- 利用目的
- 第三者提供の有無
- Cookieの使用（GA4のトラッキング）
- 問い合わせ窓口

**特定商取引法に基づく表記（フリーランスでも必須）**
- 事業者名：相川典義
- 所在地：[都道府県まで記載でも可]
- 電話番号または問い合わせフォームURL
- 販売価格：各サービスページに明記
- 支払い方法・時期
- 役務の提供時期（例：契約後14日以内）
- 返品・キャンセルポリシー

### クライアントサイトで必要なページ

フォームで個人情報を収集するすべてのサイトに必須：
- プライバシーポリシーページ（または1ページ内のセクション）
- フォームの同意チェックボックス

## クラウドワークス規約の重要ポイント

### 禁止行為（規約違反リスク）
- クラウドワークス経由で知り合ったクライアントへの直接取引誘導
- 虚偽の実績・スキルの記載
- 複数アカウントの作成
- 外部への誘導（LINE・メールへの誘導は要注意）

### 安全取引のガイドライン
- 必ず「仮払い」確認後に作業開始
- 大型案件は「マイルストーン払い」の活用
- 納品物の確認依頼は書面（チャット記録）を残す
- クライアントからの要求変更は都度チャットで記録

## トラブル対応フロー

### よくあるトラブルと初期対応

**「気に入らないので全額返金してほしい」**
1. まず冷静に「何が気に入らないか」を具体的に聞く
2. 契約書の修正範囲内であれば修正対応する
3. 契約範囲外の要求には「追加費用が発生する旨」を書面で伝える
4. 折り合いつかない場合はクラウドワークスの仲裁申請を検討

**「納期を大幅に過ぎた」**
1. 事前の進捗報告不足が原因の場合：誠実に謝罪し、確定納期を再設定
2. 状況報告のため中間納品物（ワイヤーフレーム等）を即時提出
3. 遅延損害金の定めがない場合でも誠意ある対応が最優先

**「制作物の著作権は私のものだと言っている」**
1. 契約書の著作権条項を確認
2. 入金前の場合：著作権はまだ移転していないことを伝える
3. 入金確認後であれば：著作権移転証明書を発行

## 他チームとの連携事項

- **営業チーム**：提案文の免責事項・修正条件の文言を確認
- **経理チーム**：請求書に記載する支払い条件の法的適正性を確認
- **制作チーム**：使用素材のライセンス確認を依頼される
